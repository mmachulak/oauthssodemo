The OpenID Connect library has to be configured for the application before the user can be presented with an element enticing them to sign in.

NOTICE:For Google as IDP, you first need to register your application at the Developer Console.

client_id client_id generated by the OpenID Connect provider

client_secret client_secret generated by the OpenID Connect provider

user_agent HTTP User-Agent to provide for this application

scope List of space delimited permissions the application is asking for

auth_uri URI of the Authorization endpoint at the OpenID Connect provider

token_uri URI of the Token endpoint at the OpenID Connect provider

tokeninfo_uri URI of the TokenInfo endpoint at the OpenID Connect provider

userinfo_uri URI of the UserInfo endpoint at the OpenID Connect provider

(other params) Other application parameters can be used in the constructor as well.

 

Code snippet for the setup phase:


IDP_FLOW = OpenIDConnectFlow(
    client_id = "432566608383.apps.googleusercontent.com",
    client_secret = "*************",
    user_agent = "OpenID Connect Demo - Python",
    auth_uri = "https://accounts.google.com/o/oauth2/auth",
    token_uri = "https://accounts.google.com/o/oauth2/token",
    tokeninfo_uri = "https://www.googleapis.com/oauth2/v1/tokeninfo",
    userinfo_uri = "https://www.googleapis.com/oauth2/v1/userinfo",
    access_type = "offline")

 

For your convenience, the OpenID Connect library provides default values for the Google IDP and the setup phase can be simplified as below:


IDP_FLOW = OpenIDConnectFlow(client_id='432566608383.apps.googleusercontent.com',
        client_secret='*************',
        user_agent='OpenID Connect Demo',
        scope='https://www.googleapis.com/auth/userinfo.email https://www.googleapis.com/auth/userinfo.profile',
        access_type='offline')
